# MailPilot v0.3.0

## Highlights

- Runtime parity hardened so dev mode and the installed desktop app follow the same startup, API, attachment, and full-body paths.
- Gmail account capability, reconnect, and recovery flows were stabilized with clearer frontend error handling and safer backend token handling.
- Mailbox, focus, dashboard, insights, settings, and auth surfaces received a UX polish pass for loading, empty, error, and lock/recovery states.
- Frontend API handling, Gmail MIME parsing, attachment metadata/download logic, and account capability checks were centralized to reduce fragility.
- Repo hygiene, ignore rules, formatting baselines, CI checks, and release/security documentation were tightened for safer collaboration and cleaner releases.

## User-facing improvements

- Installed app startup is more trustworthy and provides clearer recovery actions when backend readiness stalls.
- Full message body loading and attachment download behavior are more reliable in packaged mode.
- Login, local lock, and password recovery flows explain the app-local password model more clearly.
- Dashboard, focus, and insights pages handle loading and empty states more deliberately instead of looking broken or unfinished.

## Release verification

- Backend tests, frontend build, formatting/lint gates, and Tauri packaging were re-run on the release candidate.
- The MSI-installed app was smoke-tested through startup, auth, inbox/detail/full-body, attachment download, focus, dashboard, insights, views, drafts, and settings/accounts.
